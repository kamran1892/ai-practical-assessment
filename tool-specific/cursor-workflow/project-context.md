# Project Context (Cursor)

## Project

Support Ticket Management System — Core only (no Stretch)

## Stack

- Frontend: React (`project/frontend`)
- Backend: Node.js (`project/backend`)
- Database: MySQL (`project/db` + `database/`)

## In scope

- Users (seeded only)
- Tickets with status state machine
- Comments
- Keyword search + status filter
- Validation, error handling
- State-machine integration tests
- Lifecycle docs + prompt history

## Out of scope

- Auth / JWT / protected routes
- User CRUD UI
- Pagination / priority-assignee filters beyond Core
- Docker / CI / Swagger (Stretch)

## Working style

Prefer small steps, review AI output before accepting, keep secrets out of the repo.
